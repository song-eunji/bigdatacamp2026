# Spec Kit 입력 — 활동 기록 → 포트폴리오 생성기

## 1) $speckit-constitution 뒤에 붙일 원칙
- 참가자는 입력창 하나에 3줄만 적으면 그날 기록이 끝나야 한다(마찰 최소화)
- 모바일 한 손 입력 우선, 버튼/입력창 44px 이상
- 외부 API 키 없이도 로그 저장은 로컬(localStorage)만으로 데모가 동작해야 한다
- 접근성: 색만으로 정보 전달 금지
- 단일 HTML 파일 + 서버 함수 1개, 프레임워크 없이 단순하게

## 2) $speckit-specify 뒤에 붙일 한 줄 요약
매일 3줄 활동 로그를 입력하면 자동으로 누적되고, 버튼 한 번으로 그 누적 로그를 근거로 성장 스토리형 포트폴리오 초안을 생성해주는 웹앱.

## 3) 유저스토리 + 이번 범위 밖

유저스토리:
- 캠프 참가자로서 오늘 한 일을 3줄로 짧게 입력하면 자동 저장된다
- 캠프 참가자로서 지금까지 입력한 로그를 날짜별 타임라인으로 다시 볼 수 있다
- 캠프 참가자로서 버튼 하나로 누적된 로그 전체를 근거로 한 포트폴리오 초안을 받는다
- 캠프 참가자로서 초안에 어떤 로그가 근거로 쓰였는지 확인할 수 있다(신뢰 확보)
- 캠프 참가자로서 로그가 하나도 없으면 "먼저 기록을 남겨주세요" 안내를 받는다
- (확장) 캠프 참가자로서 특정 날짜의 로그를 수정/삭제할 수 있다

이번 범위 밖:
- 회원가입/로그인
- 디자인 템플릿 커스터마이징 · PDF 내보내기
- 타인과의 공유/협업 · 자동 리마인더 알림

## 4) $speckit-clarify 예상 질문 3개 + 권장 답
- 로그 저장 방식은? → localStorage(브라우저 로컬), 서버 DB 불필요 (Day1 기준. Day2에서 Supabase로 영속화 확장 가능)
- "성장 스토리" 포맷은? → 시간순 타임라인 + 한 문단 요약(어떤 역량이 어떻게 늘었는지) 조합
- 로그가 1~2개뿐일 때는? → "더 기록하면 더 풍부해져요" 안내 + 그래도 초안은 생성(빈 상태 처리)

## 5) $speckit-plan 뒤에 붙일 기술 스택 한 줄
프레임워크 없이 단일 index.html + 내장 CSS/JS. 로그는 localStorage 배열로 저장. 포트폴리오 생성은 서버 함수 `/api/portfolio`(Vercel Functions)가 OpenAI를 호출 — 키는 서버에서만 사용, 프론트에 노출 안 함.

## (참고) $speckit-checklist 예상 항목
- 로그 0개일 때 "생성" 버튼 비활성화/안내 처리
- 로그 텍스트가 공백만 있을 때 저장 막기
- 로그 1건이 너무 길 때(예: 500자↑) 처리
- 색 대비 등 접근성
- localStorage 용량 초과 시 처리

## (참고) $speckit-tasks 작업 분해
① 입력 폼(3줄 로그 입력) + localStorage 저장
② 로그 목록 → 날짜별 타임라인 렌더링
③ "포트폴리오 생성" 버튼 → 서버 함수 호출
④ 서버 함수: 누적 로그 텍스트를 프롬프트에 넣어 OpenAI 호출, 결과 반환
⑤ 결과 화면(초안 텍스트 + 근거 로그 표시)
⑥ 빈 상태/에러 상태 처리
⑦ localStorage 저장·불러오기 안정화(새로고침 유지 확인)
